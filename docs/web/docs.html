BORSUK

Documentation

Vector search APIs with explicit storage and memory behavior.

Rust is the core. Python 3.12+ uses PyO3. Node 22, 24, and 26 use N-API. The CLI is for administration, not package runtime calls.

storage Parquet segments

Vectors, sketches, routing rows, manifests, summaries, and graph blocks stay binary and columnar.

memory bounded routing

Approximate search walks from the top routing layer through selected parent pages before fetching selected leaf page objects.

api ids or vectors

Normal searches return ids or stored vectors. Report APIs add counters for tuning.

API Shape

Records are ids plus vectors. Callers may add only vectors and receive generated string ids, or add vectors with their own string, binary, or integer ids. There are no payload references in the public API. Explicit integer ids are encoded as compact unsigned varint bytes; shorter ids are preferred because storage uses compact binary ids internally.

index.add_vectors(vec![vec![1.0, 0.0], vec![0.0, 1.0]])?;
index.add_vectors_with_ids(vec![vec![0.9, 0.1]], vec!["alpha".into()])?;

let ids = index.search_ids(&[1.0, 0.0], SearchOptions::exact(10))?;
let vectors = index.search_vectors(&[1.0, 0.0], SearchOptions::exact(10))?;
let alpha = index.get_vector("alpha")?;
index.add([[1.0, 0.0], [0.0, 1.0]])
index.add([[0.9, 0.1]], ids=["alpha"])

ids = index.search_ids([1.0, 0.0], k=10)
vectors = index.search_vectors([1.0, 0.0], k=10)
alpha = index.get_vector("alpha")
await index.add([[1, 0], [0, 1]]);
await index.add([[0.9, 0.1]], ["alpha"]);

const ids = await index.searchIds([1, 0], { k: 10 });
const vectors = await index.searchVectors([1, 0], { k: 10 });
const alpha = await index.getVector("alpha");

Create-Time Parameters

uri, metric, dimensions, and segment_max_vectors define the physical index. segment_max_vectors controls L0 ingest objects. Read-optimized leaf size is controlled by compaction.

Runtime Parameters

cache_dir, open-time ram_budget, paged routing, k, search mode, leaf mode, max_segments, max_bytes, max_latency_ms, and max_candidates_per_segment are request/open knobs. Cache opens read fresh CURRENT, refetch stale active metadata cache files by checksum, and repair corrupt cached segment, graph, or routing page payloads from backing storage. Compaction is bounded by default; use compaction max_segments for batch size and rebuild only for explicit full source-level rewrites.

RAM Budget

Resident budget checks are hard failures. BORSUK reports resident_bytes_estimate; if that exceeds the configured budget, create/open/add/compact returns a RAM budget error instead of dropping segments or returning partial results.

Storage format

Binary Tables, One Tiny Pointer

Every durable table except CURRENT is Parquet. The pointer only names the active manifest version and verifies the binary metadata tables.

CURRENTbinary pointer
manifests/*.parquetconfig, levels, active refs
routing/*.parquetsummaries, pivots, blooms
segments/L*/**/*.parquetids, vectors, sketches
graphs/L*/**/*.parquetsegment-local edges

Architecture

Query Path

Fast writes append L0 blobs. In paged mode, generated-id appends read the top routing index and fill the rightmost append branch when it is readable. Compaction builds vector-local leaves and reuses unchanged routing page objects; parent binary routing pages sit above those leaves.

write_path = append L0 parquet blobs
compact_path = vector-local leaves + routing pages
query_io = routing pages + selected segment/graph parquet bytes

Search math

Routing First, Exact Rerank Last

Approximate modes change candidate generation, not result scoring. The final ranking still uses the index metric on exact vectors.

Exact segment lower bound

lb(q, s) = distance to persisted vector bounds

fallback: lb(q, s) = max(0, d(q, c_s) - r_s)

If the bound cannot beat the current kth distance, exact search skips that segment.

Approximate candidate cap

C_s = top_m delta(sketch(q), sketch(x))

m is max_candidates_per_segment. The sketch is scalar for SQ, UInt8 PQ for PQ/VamanaPQ, and graph-expanded for graph modes.

Leaf Modes

Approximate search first chooses segments, then each selected segment chooses candidate rows. Every approximate mode exact-reranks the selected rows before returning ids or vectors.

Testing and performance

Mode Evaluation

Measured with cargo run --locked --release -p borsuk --example benchmark_report on 10k synthetic vectors plus the scikit-learn digits CSV. Datasets are bulk inserted, compacted into vector-local leaves, then queried. Synthetic datasets use 64 dimensions, max_segments=8, and max_candidates_per_segment=64. Recall charts use tie-aware recall, where a different id at the same exact kth distance is accepted; id recall and termination-reason counts remain visible in the data table. The report fails if pq-scan, vamana-pq, or hybrid falls below 0.95 tie-aware recall@10.

Write path

Ingest and Compaction Throughput

The benchmark report times append-only L0 ingest separately from explicit compaction. This keeps write-path cost visible instead of hiding it inside query latency.

Dataset scaling

Records vs Recall, Latency, and Memory

Scale sweeps are generated with --synthetic-records-list. The checked-in artifact covers 10k and 100k synthetic rows, normalizes dataset family names, keeps record counts sortable, and shows whether recall, query bytes, graph bytes, resident metadata, and exact-scored rows move acceptably as datasets grow.

Release gate

Million-Vector Recall and I/O

The checked-in large-scale.csv artifact comes from the ignored million-vector gate. It compares pq-scan, vamana-pq, and hybrid at the release scale with recall, termination reason, query I/O, graph I/O, and resident metadata.

Parallel pressure

Latency, QPS, RSS, and Graph Bytes

The parallel benchmark repeats the same query batch across 1, 2, 4, and 8 worker threads. RSS is sampled from the benchmark process while the batch runs; resident_bytes_estimate is the BORSUK metadata estimate reported by each query.

Functionality

The Rust crate, Python package, and TypeScript package support create/open, generated or explicit ids, exact and approximate id/vector search, batch search, Float32 buffer inputs, get-by-id vector loading, stats, scoped compaction, full-level rebuild, obsolete segment garbage collection, metric catalogs, leaf-mode catalogs, vector distance helpers, and recall_at_k/recallAtK.

Decisions

Persistent index state is binary and columnar. CURRENT is a fixed binary pointer. Manifests, segment summaries, pivots, routing rows, vector records, scalar/PQ codes, and graph blocks are Parquet-backed binary data.
